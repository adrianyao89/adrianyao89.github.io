消息中心

支付消息发布接口

为了方便支付接口的接入，我们提供了Http的接口，用于发布支付消息。

请求地址

http://10.95.196.86/mq/paynotify

请求方式

GET

参数列表

参数名 类型 长度 必选 描述
orderid String Max(40) 非必选 第三方订单号
ordertime String Max(14) 必选 订单交易时间
recordindex String Max(20) 非必选 交易记录表主键
paytype Interger Max(3) 必选 计费接口类型，必选。0-短代，1-在线VAC，2-老版在线VAC，3-渠道支付，4-代支付， 5-短短代，6-按次代缴，7-真包月，8-代支付按次代缴，9-代支付真包月，10-cpApi支付，11-tv支付，101-支付宝， 102-神州付，103-爱游戏，104-和游戏等
cpid String Max(40) 必选 开发者/开发商ID
serviceid String Max(40) 必选 计费点ID
payfee Interger Max(10) 必选 支付金额
hret String Max(1) 必选 支付结果, 0-支付成功，1-支付失败
status String Max(10) 必选 交易状态码
usercode String Max(21) 非必选 用户手机号
provinceid String Max(8) 非必选 省份Id
cityid String Max(8) 非必选 城市Id
imsi String Max(15) 非必选 imsi号

返回

<?xml version="1.0" encoding="UTF-8"?>
<callRsp>1</callRsp>
字段 含义 类型 长度 说明
callRsp 通知处理结果 String Max(1) 0 – 错误， 1 – 正确

支付消息格式

当支付系统调用发布接口后，消息中心会把支付消息进行一些字段的补齐，然后封装为一段json字符串放入队列中。
消息接收方可以对json字符串进行解析，进行自己的逻辑。
JSON字符串如下：

字段名 类型 长度 必选 描述
orderid String Max(40) 非必选 第三方订单号
ordertime String Max(14) 必选 订单交易时间
recordindex String Max(20) 非必选 交易记录表主键
paytype Interger Max(3) 必选 计费接口类型，必选。0-短代，1-在线VAC，2-老版在线VAC，3-渠道支付，4-代支付， 5-短短代，6-按次代缴，7-真包月，8-代支付按次代缴，9-代支付真包月，10-cpApi支付，11-tv支付，101-支付宝， 102-神州付，103-爱游戏，104-和游戏等
cpid String Max(40) 必选 开发者/开发商ID
serviceid String Max(40) 必选 计费点ID
servicename String N/A 必选 道具名称
payfee Interger Max(10) 必选 支付金额
hret String Max(1) 必选 支付结果, 0-支付成功，1-支付失败
status String Max(10) 必选 交易状态码
usercode String Max(21) 非必选 用户手机号
provinceid String Max(8) 非必选 省份Id
cityid String Max(8) 非必选 城市Id
imsi String Max(15) 非必选 imsi号
appid String N/A 非必选 应用ID号
appname String N/A 非必选 应用名称
developername String N/A 非必选 开发者名称
result String N/A 必选 查询是否成功：0-是，1-否
returncode String N/A 必选 状态外码：具体错误码
appindex String N/A 必选 应用index
sdkversion String N/A 非必选 sdk版本号
vaccode String N/A 非必选 vac侧产品ID
cpparam String N/A 非必选 支付冗余字段（3G营业员编号）
bigpaytype Integer N/A 必选 大数据支付类型,0-短代 1-在线 2-代支付 3-按次代缴 4-真包月 5-渠道 6-短短代 999-不处理
subbigpaytype Integer N/A 必选 大数据支付子类型
channelid String N/A 必选 渠道id
